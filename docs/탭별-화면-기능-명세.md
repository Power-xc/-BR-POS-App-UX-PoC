# 던킨 매장 관리 앱 — 탭별 화면 & 기능 명세

> 작성 기준: BRK 백엔드 API + UX 리서치 v3.0 + 과제정의서
> 대상: 프론트엔드 구현 시 화면 상태·기능·API 연동 기준서

---

## 공통 — 앱 진입 시

### 실시간 알림 연결
- 앱 마운트 시 WebSocket `/ws/alerts` 연결
- 수신 알림 유형: `production_alert` | `order_deadline` | `system`
- 알림 수신 즉시 → 홈 탭 배지 업데이트 (24px, 검정 배경)
- 앱이 백그라운드일 때 → PUSH 알림으로 전환

### 상단 상태바 (고정)
- 매장명 + 날씨/기온 (정적, 매장 세팅에서 주입)
- 현재 시각 (1분 단위 업데이트)

## 탭 1 — 홈

> 진입 시 AI가 오늘의 핵심 상황을 먼저 보여주는 화면. 점주가 탐색하지 않아도 됨 (Zero Inbox 원칙).

### 화면 상태

| 상태 | 조건 | 표시 |
|------|------|------|
| 로딩 | 진입 직후 | 브리핑 카드 스켈레톤 3줄 |
| 정상 | 데이터 수신 완료 | 브리핑 + 신선도 보드 + 알림 |
| 긴급 | urgent 알림 1개 이상 | 상단 긴급 배너 (검정) 고정 노출 |
| 오프라인 | 네트워크 없음 | "오프라인 — 마지막 업데이트 {시각}" 토스트 |

### 구성 요소 및 기능

#### A. AI 데일리 브리핑 카드
- API: `GET /production/alerts` + 내부 매출 요약 (BRK 정의 예정)
- NLP 텍스트 1~2문장 — 그래프 절대 금지 (UT 근거: 4.2분 → 28초)
- 핵심 수치 최대 3개 (Hick's Law)
- 하단 CTA: "발주 승인 →" — Z-Pattern 시선 유도

#### B. 신선도 라이브 보드
- API: `GET /production/alerts` (5분 폴링 또는 WebSocket push)
- 폴링 주기: 5분 (BRK 스케줄러 주기와 동기)
- 표시 항목: 상품명 / 현재고 / 잔여시간 / 소진 예상 시각
- 잔여시간 바: urgent(빨강) / warning(주황) / normal(초록)
- 긴급 항목 클릭 → 생산 등록 바텀시트 열림

#### C. 생산 등록 바텀시트 (긴급 알림 액션)
- 트리거: 긴급 알림 "생산 등록" 버튼 클릭
- 표시: 현재고 / 1시간 후 예상 재고 / 4주 평균 1·2차 생산 패턴
- API: `POST /production/register` (생산 등록 완료)
- 완료 후: Undo 스낵바 8초 + 찬스로스 피드백 메시지 표시
  - 소진 전 등록: "과거 동 시간 찬스 로스 10% 감소"
  - 소진 후 등록: "10% 찬스 로스 발생"

#### D. 우선순위 알림 목록
- API: `GET /production/alerts` 결과 중 알림 항목
- 신호등 3단계: 긴급(검정) / 주의(회색) / 정보(연회색)
- 읽지 않은 긴급·주의만 기본 노출, 정보는 접힘
- 원터치 액션 버튼 → 해당 탭으로 딥링크

## 탭 2 — AI 발주

> 주문 누락 방지가 목적. 최종 결정은 무조건 점주.

### 화면 상태

| 상태 | 조건 | 표시 |
|------|------|------|
| 로딩 | 진입 직후 | 아이템 카드 스켈레톤 |
| 정상 | 옵션 수신 완료 | 발주 아이템 목록 |
| 마감 임박 | 마감 20분 이내 | 상단 타이머 배너 (D-20분) |
| 마감 초과 | 마감 시간 경과 | 해당 그룹 비활성 + "마감 완료" 표시 |
| 승인 완료 | 일괄 승인 후 | 완료 상태 + Undo 8초 스낵바 |

### 구성 요소 및 기능

#### A. 마감 임박 배너 (조건부)
- 트리거: WebSocket `order_deadline` 알림 수신
- 표시: "베이커리 발주 마감 {N}분 전"
- 타이머 카운트다운 (1분 단위)

#### B. 발주 아이템 카드
- API: `GET /order/options/{store_id}` — 전주/전전주/전월 동요일 3가지 옵션
- 각 카드 표시 항목:
  - 상품명
  - AI 점수 (0–100, 시각적 바)
  - **AI 근거 — 점주 언어로 변환 필수** (UT 핵심)
    - X: "기상 데이터 기반 예측"
    - O: "내일 비 예보 → 실내 수요↑"
  - 수동 수량 조절 스테퍼 (점주 최종 결정권)
  - 전주/전전주 비교 (토글 상세)

#### C. 특이사항 안내 (옵션 선택 시)
- API: `GET /order/notice/{option_id}`
- 이벤트/캠페인 영향 여부 감지 → 안내 문구 자동 생성
- 예: "전주 주문의 경우 캠페인으로 4주 평균 대비 10% 많았습니다"
- 점주가 수량 수정 가능 → 수정 후 재계산

#### D. 일괄 승인 CTA
- Fitts's Law: h-14 (56px) 전폭 블랙 버튼
- API: `POST /order/confirm`
- 완료: Undo 8초 스낵바 (확인 팝업 금지)
- Undo 실행 시: 승인 상태 롤백, 재편집 가능

## 탭 3 — AI 비서

> 자연어 명령 → AI가 발주서·분석 결과를 카드로 변환. 점주는 확인·수정·승인만.

### 화면 상태

| 상태 | 조건 | 표시 |
|------|------|------|
| 초기 | 대화 없음 | 빈 채팅 + 퀵 질문 3개 |
| 응답 대기 | 메시지 전송 후 | AI 버블 로딩 애니메이션 (점 3개) |
| 결과 수신 | 응답 완료 | 텍스트 또는 인라인 카드 |
| 에러 | API 실패 | "잠시 후 다시 시도해 주세요" 버블 |
| 발주서 생성 | draft 포함 응답 | 인라인 발주서 카드 + 승인 버튼 |

### 구성 요소 및 기능

#### A. 채팅 인터페이스
- API: `POST /chat` (스트리밍 응답 — SSE 또는 청크 수신)
- 입력: 텍스트 (엔터 전송)
- 퀵 버튼: "재고 얼마나 남았어?" / "이번 주 매출 알려줘" / "발주서 작성해줘"

#### B. AI 응답 유형별 렌더링

| 응답 유형 | 렌더링 |
|-----------|--------|
| 텍스트 요약 | 일반 AI 버블 |
| 발주서 초안 | 인라인 발주서 카드 (아이템 목록 + 총액 + 승인 버튼) |
| 매출 비교 | 텍스트 인사이트 카드 (수치 강조) |
| 재고 현황 | 미니 신선도 카드 |

#### C. 발주서 초안 카드
- 아이템별 수량 인라인 수정 가능
- 승인 API: `POST /order/confirm`
- 승인 후: Undo 8초 스낵바

#### D. 매출 분석 질의 (Agent C 연동)
- 자연어 → `POST /chat` → 의도 분류 → SQL/분석 결과 → 인사이트 텍스트
- 단순 요약 금지, 실행 가능한 인사이트로 변환 (요구사항)
- 예상 질의 커버 범위:
  - 전년 동월 매출 비교
  - 배달 채널별(쿠팡이츠/배민/해피오더) 매출
  - T데이 실적 비교
  - 특정 상품 전월/전년 대비
  - 내 매장 vs 가맹점 평균

## 탭 4 — 리뷰

> 리뷰 센티먼트 자동 분류 + 키워드 추출. 점주가 직접 읽지 않아도 됨.

### 화면 상태

| 상태 | 조건 | 표시 |
|------|------|------|
| 로딩 | 진입 직후 | 점수 + 바 스켈레톤 |
| 정상 | 데이터 수신 | 점수 + 분포 + 키워드 + 최근 리뷰 |
| 데이터 없음 | 리뷰 0건 | "아직 리뷰가 없습니다" 빈 상태 |

### 구성 요소 및 기능

#### A. 종합 점수 & 분포
- API: BRK 정의 예정 (리뷰 분석 엔드포인트)
- 점수: X.X / 5.0
- 분포 바: 긍정(검정) / 중립(회색) / 부정(연회색)
- 전주 대비 트렌드 (↑/↓/-)

#### B. 키워드 클러스터
- 긍정 키워드: 초록 배경 태그
- 개선 필요 키워드: 빨강 배경 태그
- 키워드 클릭 → 해당 키워드 포함 리뷰 필터링

#### C. 최근 리뷰 목록
- 별점 + 센티먼트 라벨 + 날짜 + 본문
- 배달 채널 출처 표시 (쿠팡이츠/배민/해피오더/매장)
- 무한 스크롤 (페이지네이션)

## POS 화면 (`/pos`)

> 태블릿 전용. 피크타임 화면 전환 없이 주문·대기열 동시 처리.

### 화면 상태

| 상태 | 조건 | 표시 |
|------|------|------|
| 정상 | 운영 중 | 메뉴 그리드 + 우측 패널 |
| 품절 처리 | 품절 버튼 클릭 | Undo 스낵바 8초 |
| 결제 완료 | 결제 처리 후 | Undo 스낵바 8초 + 대기열 자동 추가 |
| 긴급 알림 | WebSocket 수신 | 우측 패널 상단 배너 (탭 전환 불필요) |

### 구성 요소 및 기능

#### A. 메뉴 그리드 (좌측)
- 카테고리 탭 4개 이내 (Hick's Law)
- 상품 카드: 이름 + 가격 + 품절 여부
- 원터치 장바구니 추가
- 품절 처리: 클릭 즉시 → Undo 8초 스낵바

#### B. 우측 패널 탭 [장바구니|대기열] — V3 확정
- UT 결과: 시선 이동 85px(↓80%), 성공률 94%
- 장바구니: 수량 조절 + 결제 CTA (간편결제/카드)
- 대기열: 제조중/픽업대기 그룹 분리, 배지 상시 노출
- 탭 배지: 24px, 검정 배경 (UT 피드백)
- 상태 전환: 제조중→픽업완료 원터치 (확인 팝업 없음)

## 프론트엔드 구현 우선순위

```
P0 (데모 필수)
├── WebSocket 실시간 알림 연결
├── 홈 — 생산 등록 바텀시트 + 찬스로스 피드백
├── AI발주 — 실 API 연동 + 특이사항 카드
└── AI비서 — POST /chat 스트리밍 연동

P1 (POC 완성도)
├── 마감 임박 타이머 배너
├── 리뷰 실 API 연동
└── POS — WebSocket 알림 인라인 배너

P2 (고도화)
├── 오프라인 대응
├── 무한 스크롤
└── 키워드 클릭 필터링
```
