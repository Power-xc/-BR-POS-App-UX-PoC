# 던킨 POC 기획 변경 — 2026-04-06 회의 반영

> 출처: BR Korea 던킨 PoC 회의록
> 적용 기준: 즉시 (POC 범위 확정)

---

## 핵심 변경 요약

| 항목 | 이전 | 변경 후 |
|------|------|---------|
| POS 타입 | 포스기 (하드웨어) | **웹 POS** — 내부 구성만 변경 가능 |
| KDS | 포함 | **전면 제외** — 현재 미운영 |
| 키오스크 연동 | 고도화 후 포함 | **전면 제외** — 관리 부서 다름 |
| POS 외부 인터페이스 | 자유 변경 가정 | **변경 불가** — 내부 구성만 자유 |
| 모바일 POS | 일반 | 설계 **자유도 높게** 진행 |
| 실시간 재고 API | 있다고 가정 | **현재 불가** — 엑셀 데이터, 분단위 API는 정식 전환 후 |
| POC 일정 | 원래 일정 | **1주 연장** |

---

## 1. 웹 POS — 제약 사항 확정

```
✅ 가능: 내부 구성 변경 (데이터 표시 방식, AI 기능 삽입, 패널 추가)
❌ 불가: 화면 자체(외부 인터페이스) 변경 — 기존 POS 화면 틀 유지 필수
```

**영향:** 현재 `src/app/pos/page.tsx`는 **내부 AI 기능 POC 데모**로 유지.
실제 웹 POS에 붙일 때는 외부 레이아웃 변경 없이 내부 패널/데이터만 교체.

---

## 2. KDS — 완전 제외

- **이유:** 던킨 강남본점 기준 KDS 현재 미운영
- **영향:** 마인드맵 `대기·KDS` 섹션 → 웹 POS 내 대기열 UI로만 표현 (이미 구현됨)
- **현재 구현 (`pos/page.tsx` 대기열 패널):** 그대로 유지 — KDS가 아닌 "웹 POS 내 주문 대기열"로 포지셔닝

---

## 3. 키오스크 — 완전 제외

- **이유:** 관리 부서가 POS와 다름. 현 단계에서 연동 범위 외
- **영향:**
  - `docs/디자인-시스템-계획.md` 내 키오스크 관련 항목 무시
  - 마인드맵 "POC 후 고도화 — 키오스크 연계" 섹션 폐기
  - 하드웨어 확장(카메라 등)은 *POC 이후 별도 논의*

---

## 4. 생산 관리 Agent — 운영 방식 확정

| 항목 | 확정 내용 |
|------|---------|
| 생산 등록 시점 | 생산 **완료** 시점 (시작 시점 X) |
| 등록 주체 | 점주 또는 알바 직접 입력 |
| 리드타임 기준 | 품목·점포마다 상이 → POC에서는 **1시간** 통일 기준 |
| 찬스로스 기준 | 내부 기준 없음 → **역추정 방식** (과거 판매 데이터 기반) |

---

## 5. 주문 관리 Agent — 승인 3단계 확정

```
본사 승인 → AI 승인 → 룩 확정
```

- 긴급 발주 별도 프로세스 없음 → AI가 부족 감지 + 보충 필요성 제안하는 것으로 대체

---

## 6. 데이터 환경

| 항목 | 현황 |
|------|------|
| 실시간 재고 API | **없음** — 타임스탬프 기반 실시간 불가 |
| 정식 전환 후 | 분 단위 재고 전송 API 제공 예정 |
| POC 데이터 | 엑셀 파일 수령 (화면 접근 권한 미확보 → 화면 별도 제공 예정) |
| 현재 mock 방식 | 엑셀 기반 정적 mock → 추후 API 연결 |

---

## 7. 모바일 — 방향 확정

- 점주 입장의 기존 화면 연속성 유지 **또는** 새 방향성 검토 모두 가능
- 설계 자유도 높게 진행
- 고령 점주 접근성 고려 (이미 폰트 +1 step 반영 완료)

---

## 8. 현재 POC 구현체 포지셔닝

| 구현 화면 | 포지셔닝 |
|---------|---------|

- 마케팅 데이터 활용
- CJ 등 타 업체 사업 확장 가능성
- 키오스크 연계 → 별도 프로젝트로 분리
