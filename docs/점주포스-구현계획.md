## 1. 배경 & 포지셔닝

```
기존 웹 POS (비알코리아 시스템)
└── [점주 포스 이동] 버튼 추가 (내부 구성 자유)
        └── /pos 경로 → 점주 포스 (완전 새 설계)
                        주문·결제 없음. AI 기반 생산·발주·매출 관리만.
```

현재 `pos/page.tsx`는 판매용 POS(장바구니·대기열·결제) 구조 → **완전 교체.**

## 4. 탭별 구현 명세

### 4-1. 홈·현황 (HomeTab)

**구성 요소:**

```
AI 데일리 브리핑 헤더 (에이전트 C 뱃지 + 날짜)
└── 다크 카드: AI 브리핑 요약 텍스트

4가지 메트릭 카드 (2×2 그리드):
├── 오늘 매출 (전일 대비 % 표시)
├── 기회손실 추정 (개선 필요 강조)
├── 폐기 비용 (전일 대비 % 표시)
└── 즉시 조치 필요 (건수 + "생산 부족" 태그)

즉시 조치 필요 섹션 (에이전트 A 감지 태그):
└── 품목 카드 × N: 품목명 + 이유 + [생산 관리 >] 버튼

하단 퀵 네비 카드 (3분할):
├── 생산 관리 (조치 필요 건수)
├── 발주 관리 (3단계 승인 대기)
└── 매출 분석 (AI 브리핑 보기)
```

### 4-2. 재고·생산 (InventoryTab)

**구성 요소:**

```
상단 3개 요약 카드:
├── 현재 기회손실 추정 금액 (역추정 기준 리드타임 1h)
├── 즉시 생산 필요 품목 수 (에이전트 A 감지)
└── AI 근거 보기 (역추정 공식 설명 토글)

품목 카드 리스트:
└── 각 카드:
    ├── 품목명 + 상태 뱃지 (즉시 생산 / 생산 권장 / 적정)
    ├── AI 이유 텍스트
    ├── 현재 재고 → 권장 생산 수량
    ├── 기회손실 금액 (해당 시)
    └── [생산 완료 등록] 버튼 (즉시 생산 / 생산 권장 시)
```

**상태 뱃지 기준:**

| 뱃지 | 기준 | 색상 |
|------|-----|------|
| 즉시 생산 | 재고 소진 예정 ≤ 30분 | `primary` (black) |
| 생산 권장 | 30분 < 소진 예정 ≤ 1시간 | `warning` (orange) |
| 적정 | 소진 예정 > 1시간 | `secondary` (gray) |

**[생산 완료 등록] 클릭 시:**
- Snackbar: "글레이즈드 생산 완료 등록됨 · 실행 취소" (8초)
- 상태 → 적정으로 전환 (낙관적 업데이트)

### 4-3. 발주 관리 (OrderTab) — 3단계 위자드

**Stepper 컴포넌트 활용** (`src/shared/ui/Stepper.tsx` 기존 컴포넌트)

```
단계 1 — 본사 지시 확인 (Step1HqInstruction)
  → 본사 발주 지시 테이블 (품목 / 본사 지시 수량 / 비고)
  → [확인 — AI 분석으로 넘기기 >] 버튼

단계 2 — AI 검토 (Step2AiReview)
  → AI 분석 결과 테이블 (품목 / 본사 / AI 제안 / 변경 이유)
  → 변경된 항목: [변경] 뱃지 + 굵은 숫자
  → [← 이전] [AI 추천 수량으로 최종 확인 >]

단계 3 — 점주 최종 확정 (Step3FinalConfirm)
  → 최종 수량 카드 리스트 (각 항목 인라인 수량 수정 ✏️)
  → [← 이전] [발주 최종 확정 전송 ⚡]

완료 화면
  → ✓ 발주 확정 완료 + 시간 + 이력 저장됨
  → [새 발주 시작] [홈으로]
```

### 5-1. Agent A: 재고 역추정 로직

**핵심 철학:** 실시간 재고 API 부재 → POS 결제 데이터 기반 역추정으로 대체

**현재 재고 역추정 공식:**
```
현재 역추정 재고 = (오픈 시점 기초 재고 + 당일 생산 완료 누적량) - 당일 POS 결제(판매) 수량
```

**기초 재고 입력 방식:**
- 전날 마감 시 점주(또는 알바)가 직접 품목별 실사 재고 입력
- 입력값이 다음날 오픈 시점에 자동 이월
- 입력 누락 시: 전일 역추정 마감 재고를 기본값으로 사용 + 상단 경고 안내 표시
  ```
  "오늘 기초 재고가 수동 입력되지 않았습니다. 역추정 값으로 대체됩니다."
  ```

**리드타임 반영:**
- 생산 완료 등록 시점 기준 **+1시간 후** 재고에 반영 (POC 통일 기준)
- 추후 품목별 리드타임 테이블 확보 시 실제 소요 시간으로 대체 예정
- UI 명시: "생산 완료 등록 시 1시간 후 재고 반영"

**권장 생산량 공식:**
```
권장 생산량 = 해당 시간대 예상 수요량 - 현재 역추정 재고
해당 시간대 예상 수요량 = 최근 4주 동일 요일·시간대 평균 판매량
```

**기회손실 금액 공식:**
```
기회손실 금액 = 권장 생산량(부족분) × 품목 판매 단가
```

> Agent A 품목별 기회손실 합산 → Agent C 홈·현황 "기회손실 추정" 카드에 실시간 반영.
> 생산 완료 등록 시 → 기회손실 금액 자동 감소.

**AI 근거 보기 버튼 노출 텍스트 예시:**
```
역추정 공식: (기초 재고 50개 + 생산 완료 0개) - 판매 46개 = 현재 재고 4개
예상 수요 (10~11시 피크): 최근 4주 평균 30개
권장 생산: 30개 - 4개 = 26개 (+ 피크 보정 +2개 = 28개)
기회손실: 28개 × ₩1,800(판매가) = ₩50,400 → 피크 가중 ₩86,000
```

**상태 뱃지 분류:**

| 뱃지 | 기준 | 색상 |
|------|-----|------|
| 즉시 생산 | 역추정 재고 ÷ 시간당 수요 ≤ 30분 | `primary` (black) |
| 생산 권장 | 30분 < 소진 예정 ≤ 60분 | 주황 |
| 적정 | 소진 예정 > 60분 | 회색 |

**POC mock 품목 기준값:**

| 품목 | 기초 재고 | 시간당 평균 판매 | 판매 단가 |
|------|---------|--------------|---------|
| 글레이즈드 | 50개 | 피크 7개/h | ₩1,800 |

### 5-2. Agent B: AI 발주량 산정 로직

**핵심 철학:** 본사 하향식 지시 수량을 매장 고유 데이터로 보정

**AI 제안 발주량 공식:**
```
AI 제안 = 본사 지시 수량 + α + β

α = 전주 동요일 동시간대 품절로 인한 기회손실 수량 (이력 기반)
β = 요일·기상·이벤트 가중치 반영 추가 수량
```

**α (기회손실 이력) 정의:**
```
α = Agent A 기회손실 이력에서 자동 산출
  = 직전 1주일 동일 요일·시간대 품절 발생 품목의 미판매 추정 수량
```

**β (이벤트 가중치) 상세:**

| 변수 | 조건 | 가중치 | 데이터 출처 |
|-----|-----|------|-----------|
| 요일 | 토·일 | +10~15% | 최근 4주 주말/평일 판매 차이 비율 |
| 이벤트 | 본사 공식 이벤트 (T데이, 시즌 프로모션) | +20~30% | 본사 이벤트 캘린더 (미제공 시 전년 동기 실적 대체) |
| 날씨 | 강수 확률 60% 이상 | 음료 +10%, 도넛 -5% | 기상청 API (연동 불가 시 미적용) |

**변경 이유 텍스트 생성 규칙 (Rule-based → LLM 연결 예정):**
- α 반영 → "기회손실 역추정 +N개 / [요일] 수요 반영"
- β 요일만 → "[요일] 패턴 반영 +N개"
- 변경 없음 → "본사 지시와 동일 — 적정 수량"
- 가장 높은 가중치 요인 1개를 우선 표기

**POC mock (7월 3주차 화요일 기준):**

| 품목 | 본사 | AI 제안 | 변경 이유 |
|------|-----|--------|---------|
| 글레이즈드 | 48개 | 56개 | 기회손실 역추정 +8개 / 주말 수요 반영 |
| 보스턴크림 | 24개 | 24개 | 본사 지시와 동일 — 적정 수량 |
| 초코링 | 30개 | 36개 | 오후 피크 기회손실 패턴 반영 +6개 |
| 아메리카노 원두 | 2kg | 2.5kg | 금·토 라시 패턴 반영 +0.5kg |
| 먼치킨 | 60개 | 60개 | 본사 지시와 동일 — 적정 수량 |

### 5-3. Agent C: 매출 분석 및 폐기 비용 로직

**오늘 매출:**
```
오늘 매출 = Σ(품목별 POS 결제 수량 × 판매 단가)
전일 대비 % = (오늘 매출 - 전일 매출) / 전일 매출 × 100
```

**기회손실 추정 (Agent A → Agent C 연동):**
```
기회손실 합산 = Σ(Agent A 품목별 기회손실 금액)
```
- 홈·현황 카드에 합산 표시: "기회손실 추정 ₩155,000 / 피크 품절 기준"
- 카드 탭 → 품목별 기회손실 상세 드릴다운
- 생산 등록 완료 시 → 기회손실 금액 자동 감소 반영

**폐기 비용 (핵심: 판매가가 아닌 원가 기준):**
```
폐기 비용 = 마감 시점 미판매 역추정 재고 수량 × 품목별 원가(공급가)
```
- 손실 관점으로 접근 → 다음날 발주·생산량 조절 근거로 활용
- 원가 데이터: 비알코리아 제공 기준 적용
- POC 중 미제공 시: 판매가 × 40% 임시 적용 + 화면 안내 표시
  ```
  "원가 데이터 미연동 상태입니다. 임시 기준값(판매가의 40%)으로 산출됩니다."
  ```

**시간대별 판매량 차트:**
- 8-19시 각 시간대별 POS 거래 집계
- 기회손실 발생 구간 주석 표시 (10-11시 글레이즈드 품절)
- 차트 구현: 순수 CSS 바 차트 (높이 = 해당 시간 판매량 / 최대 시간 판매량 × 100%)

### 5-4. 공지 우선순위 분류 기준 (Agent C)

**키워드 기반 자동 분류:**

| 등급 | 분류 기준 키워드 | 특성 |
|-----|--------------|-----|
| 긴급 | 즉시, 비활성화, 중단, 오늘 마감, 오늘까지 | 행동 촉구형. 카드 기본 펼침 + 필요 조치 텍스트 표시 |
| 주의 | 프로모션 변경, 레시피 변경, 주의, 마감 임박 | 영업 직접 영향. 카드 접힘 기본 |
| 정보 | 그 외 (시스템 점검, 교육, 일반 안내) | 참고용. 하단 "N건 더 보기"로 묶음 |

**AI 요약 형식 (POC: 정적 텍스트, 정식: LLM 생성):**
```
원문 공지 → "필요 조치: [한 줄 액션 텍스트]"
예: "오늘(7/14) 마지막 판매일. 내일(7/15)부터 메뉴에서 제거 필요."
    → 필요 조치: 오늘 판매 후 메뉴 비활성화
```

**동일 키워드 복수 등급 해당 시:** 상위 등급 우선 적용

**이미지 공지 처리:**
- OCR 처리 후 텍스트 추출 → 동일 분류 기준 적용
- OCR 불가 시: '정보' 등급 분류 + 원문 이미지 그대로 노출

**점주 포스 진입 시 긴급 건 최상단 배치 → 즉시 조치 버튼 제공**
